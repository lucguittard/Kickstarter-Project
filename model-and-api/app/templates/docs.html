Model API Documentation

The model API receives a JSON request containing model features:

  • backers_count - number of backers
  • goal - the amount of money being requested by the campaign
  • spotlight - whether the campaign has spotlight status
  • staff_pick - whether the campaign is a staff pick

The model API returns a JSON response containing:

  • ESTIMATE - the comparative success of a campaign relative to other campaigns; ranging from 10 to 1, with a 10 signifying poor performance (in the tenth decile) and a 1 signifying best performance (in the first decile).
  • MISSING_DATA - whether or not the received request was missing any feature data
